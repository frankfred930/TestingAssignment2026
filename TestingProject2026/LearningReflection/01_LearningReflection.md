## 1. How was your experience testing the given Webapp?
 
    It was both challenging and educational. Testing a real-world application was valuable because it exposed edge cases that textbook examples never show.


## 2. How did you write or manage your test case? Describe the process.
 
    I followed a structured approach:

    Step 1: Understand the template format

    Step 2: Analyze each page's functionality

    Step 3: Write test cases using the Given-When-Then pattern

    Step 4: Separate Positive and Negative tests

    Step 5: Define reusable variables at the top for test data


## 3. Do you recommend any other tools or styles for Test case management. 
      
    Yes, I recommend Gherkin because it's readable by non-technical stakeholders and it forces clear thinking about preconditions and expected outcomes.


## 4. Which IDE (Visual Code or Atom or else) have you used to edit files?
 
    I used Visual Studio Code because:

    Lightweight and fast

    Syntax highlighting for .txt and .md files

    Easy file navigation with the explorer panel

     
## 5. Did you find any trouble? how did you solve the trouble?

    The biggest trouble was understanding the precondition chain: You cannot test issues without a project, and you cannot test projects without logging in. I solved this by documenting preconditions clearly and testing 
    
    in sequence: Login → Create Project → Create Issue.
